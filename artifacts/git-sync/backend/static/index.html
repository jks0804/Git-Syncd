Webhook Setup

Paste this URL into your GitHub or GitLab repository's webhook settings. When a push event fires, it will automatically trigger a sync.

Webhook URL
Secret Key (recommended — validates the request is from your git host)
  1. Go to your GitHub repo → Settings → Webhooks → Add webhook
  2. Paste the Webhook URL above into the Payload URL field
  3. Set Content type to application/json
  4. Paste the Secret Key above into the Secret field
  5. Choose Just the push event (or "Send me everything")
  6. Check Active and click Add webhook
GitSyncd validates the X-Hub-Signature-256 header using HMAC-SHA256.
  1. Go to your GitLab repo → Settings → Webhooks
  2. Paste the Webhook URL above into the URL field
  3. Paste the Secret Key above into the Secret token field
  4. Check Push events
  5. Click Add webhook
GitSyncd validates the X-Gitlab-Token header.

Send a POST request to the webhook URL. Include the secret in one of these headers:

X-Hub-Signature-256: sha256=<hmac-sha256-of-body> ← GitHub style X-Gitlab-Token: <your-secret> ← GitLab style X-Webhook-Secret: <your-secret> ← Generic

If no secret is set, all POST requests trigger a sync.
